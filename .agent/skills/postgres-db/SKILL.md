## 🧠 Reglas de Oro para PostgreSQL (Node.js Driver)

1.  **Bind Variables (SEGURIDAD)**:
    * **OBLIGATORIO**: Jamás concatenes valores.
    * Usa el signo peso y un número secuencial: `$1`, `$2`, `$3`.
    * *Ejemplo*: `SELECT * FROM USUARIOS WHERE RUT_ANALISTA = $1`

2.  **Manejo de Tipos de Dato**:
    * **Booleanos**: PostgreSQL tiene tipo nativo `BOOLEAN`.
        * App (`true`) -> DB (`true`)
        * App (`false`) -> DB (`false`)
        * *Nota*: Ya no uses `0` o `1` como en Oracle.
    * **Fechas**:
        * Postgres maneja nativamente strings ISO 8601 (`YYYY-MM-DD`).
        * No necesitas `TO_DATE`. Simplemente pasa el string: `VALUES ($1)` donde `$1` es `'2026-01-21'`.
    * **IDs Autoincrementales**:
        * Para obtener el ID recién creado después de un insert, usa `RETURNING`.
        * *Ejemplo*: `INSERT INTO ... VALUES (...) RETURNING ID_SESION;`

3.  **Configuración de Salida**:
    * La librería `pg` devuelve un objeto resultado. Las filas están en la propiedad `.rows`.
    * `const res = await pool.query(...)` -> Los datos están en `res.rows` (Array de objetos).

### 4. Maestros y Catálogos (Lookups)
Tablas maestras para llenar selects en el Frontend. IDs son `INT` (SERIAL).

* **`EQUIPOS_LAB`**: `ID_EQUIPO`, `NOMBRE_EQUIPO`, `CODIGO_EQUIPO`
* **`MICROPIPETAS`**: `ID_PIPETA`, `NOMBRE_PIPETA`, `CAPACIDAD`
* **`INSTRUMENTOS`**: `ID_INSTRUMENTO`, `NOMBRE_INSTRUMENTO`
* **`DILUYENTES`**: `ID_DILUYENTE`, `NOMBRE_DILUYENTE`
* **`MATERIAL_SIEMBRA`**: `ID_MATERIAL_SIEMBRA`, `NOMBRE_MATERIAL`
* **`MAESTRO_CHECKLIST_LIMPIEZA`**: Items por defecto.

### 5. Vistas Útiles
**`V_CATALOGO_UNIFICADO`**:
* Consulta unificada de `INSTRUMENTOS` y `MICROPIPETAS`.
* **Columnas**: `id_origen`, `nombre`, `codigo`, `tipo_material` ('INSTRUMENTO' | 'PIPETA'), `capacidad`.
